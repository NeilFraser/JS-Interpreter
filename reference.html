JS-Interpreter Reference Manual

JS-Interpreter can be used with as little as the constructor, the run function, and the value property:

var myInterpreter = new Interpreter('6 * 7');
myInterpreter.run();
alert(myInterpreter.value);

However, if one wants to create APIs for the interpreted code to call, tighter control of how execution runs, or multi-threading, then the reference material below will be needed.

Constructor

Interpreter(code, opt_initFunc)
Creates a new JS-Intepreter instance, ready to execute the provided code.
  • code -- The JavaScript code to execute. Either a string of code, or (unusually) a pre-parsed AST object.
  • opt_initFunc -- A function which (if present) is called to allow API methods to be added to the new interpreter. This function is called with two arguments, the interpreter instance, and its global object.
Example: 
var initFunc = function(interpreter, globalObject) {
  var wrapper = function alert(text) {
    return window.alert(text);
  };
  interpreter.setProperty(globalObject, 'alert',
      interpreter.createNativeFunction(wrapper));
};
myInterpreter = new Interpreter('alert(6 * 7)', initFunc);

Instance properties

value
Stores the final computed value, similar to the eval function.
Example: 
var myInterpreter = new Interpreter('6 * 7');
myInterpreter.run();
alert(myInterpreter.value);

Instance methods

step()
Executes the interpreter by a single step. Returns true if there are additional steps to execute, false if execution is complete and no tasks are pending.
Example: 
function nextStep() {
  if (myInterpreter.step()) {
    window.setTimeout(nextStep, 0);
  }
}
nextStep();
run()
Executes the interpreter until it runs out of code (as indicated by a return value of false), or until an asynchonous function pauses execution (as indicated by a return value of true). This function is not recommended if infinite loops are possible, or setTimeout/setInterval are present.
Example: 
myInterpreter.run();
getStatus()
Returns the current status of the interpreter. See the Status section for details. Usage: 
if (myInterpreter.getStatus() === Interpreter.Status.DONE) {...}
appendCode()
Add more code to the interpreter. This code is appended to the code provided in the constructor, or appended to the code provided by previous calls to appendCode. Code may be added at any time, even after the status reports "DONE".
  • code -- The JavaScript code to execute. Either a string of code, or (unusually) a pre-parsed AST object.
Example: 
myInterpreter.appendCode('foo();');

Static properties


  

    
Status

    
A map of possible output values for the getStatus function.
    Possible values are DONE, STEP, TASK,
    or ASYNC.


    
VALUE_IN_DESCRIPTOR

    
 Unique symbol for indicating, when used as the value of the 'value'
    parameter in calls to setProperty, that the value should be
    taken from the property descriptor instead.
